# 2026Robocon-Volley-R1
**学校**：河北科技大学

**项目名称**：2026Robocon-Volley-R1（排球机器人）

---

## 一、项目核心配置

### 1.1 主控单元
采用 **STM32F407VET6** 作为主控板，负责机器人所有运动控制、传感器数据处理、执行器驱动及任务调度，具备高性能、高稳定性及丰富的外设接口，满足机器人实时控制需求。

### 1.2 底盘系统
- 驱动方案：采用 **VESC电调** 驱动 **5055电机**，电机布局为 **等边三角形**，实现全向移动，兼顾灵活性与稳定性。
- 控制参数（上位机电流环）：
  - 比例系数 Kp = 0.1
  - 积分系数 Ki = 200.0
- 打滑矫正：引入 **JY61陀螺仪**，解决全向轮底盘快速侧移时的滑移与偏航问题。

### 1.3 击球装置
**版本迭代说明**：
- 初始方案（作废）：采用双 **灵足04电机** 驱动击球装置，因力矩输出及机械结构存在局限性，无法满足排球击球力度与精度要求，故废弃。
- 最终方案：改为 **三气缸控制**，优化击球力度调节范围，提升击球稳定性与响应速度，适配排球竞赛场景。

### 1.4 遥控系统
遥控器主控：**ESP32-S3**，负责指令采集、编码与发送，具备低功耗、强抗干扰能力。

**无线模块迭代说明**：
- 初始方案（作废）：采用 EBYTE E32 无线模块，因通信距离无法满足竞赛场地需求，故废弃。
- 最终方案：更换为 **HC12 无线模块**，保障远距离、稳定的指令传输，适配竞赛场地规模。

## 二、核心技术难点及解决方案

### 2.1 全向轮底盘打滑问题
**问题现象**：全向轮底盘在快速侧移时，因侧向加速度较大，轮子滚子与地面摩擦力不足，会产生瞬时滑移及额外扭矩，导致机器人轻微偏航；而缓慢移动时加速度较小，摩擦力充足，该现象不明显，最终导致运动轨迹偏离预期。

**解决方案**：引入 **JY61陀螺仪** 实时采集机器人姿态数据（角速度、角度），通过姿态闭环控制算法，动态修正电机转速，补偿滑移带来的偏航误差，实现打滑自动矫正，确保机器人运动轨迹精准。

## 三、软件架构与文件说明

### 3.1 任务调度流程
采用 FreeRTOS 实时操作系统，任务调度入口流程如下：
`main.c → freertos.c → Task_Init.c`（启动任务调度，分配各任务优先级、堆栈大小，实现多任务并行执行）

### 3.2 核心文件功能说明
| 文件名 | 核心功能 |
| ---- | ---- |
| chassis.c | 底盘运动解算核心文件，集成遥控器指令解析与力矩环控制，实现全向移动控制及打滑矫正逻辑。 |
| hitball.c | 自动回球装置控制文件，实现三气缸击球的时序控制、力度调节，完成自动回球功能。 |
| My_list.c / comm.c / dataFrame0.c(data_poll.c / comm_stm32_hal_middle.c) | 遥控器（ESP32-S3）相关通信文件，负责无线指令的接收、解析、校验，实现主控与遥控器的双向通信。 |
| step.c | 三次多项式算法实现文件，起初用于配合双电机击球装置（RobStride.c），实现电机转速平滑过渡以达到最大速度；目前保留算法，可用于后续功能扩展。 |
| CANDrive.c | CAN总线通信核心文件，负责主控与VESC电调、其他外设的CAN通信，实现电机转速、电流等数据的传输与控制指令下发。 |

## 四、项目核心功能
- 远程遥控：通过 ESP32-S3 遥控器 + HC12 无线模块，实现机器人全向移动、击球动作的远程精准控制。
- 自动打滑矫正：基于 JY61陀螺仪 姿态反馈，动态修正底盘运动轨迹，解决快速侧移时的滑移偏航问题。
- 自动回球：由三气缸击球装置配合位置检测，实现排球的自动接收与回球动作，适配竞赛场景。
